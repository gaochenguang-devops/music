# 歌词与控制布局修复设计

## 问题

歌词按钮使用 `Ui::add_sized` 和 `Button::wrap`，但歌词滚动区没有显式固定内容宽度。长文本或双语文本仍可能扩大控件的期望尺寸，造成文本越过中间区域。底栏所有控件位于同一个水平布局中，左右控件宽度不相等，因此播放按钮组无法位于窗口的几何中心。播放列表始终占据右侧空间，用户无法把空间让给歌词。

## 方案

歌词不再使用按钮隐式排版。每行先用 egui `Painter::layout` 在明确的最大宽度内生成 Galley，再按 Galley 实际高度分配整行矩形，并把文字绘制在矩形中心。整个矩形保留点击跳转和自动滚动响应。

底栏在进度条下方使用三个等宽区域：播放模式与倍速位于左区，上一曲、播放/暂停、停止、下一曲位于中区，静音和音量位于右区。中区内部使用固定尺寸按钮，因此图标和按钮组均稳定居中。

右侧播放列表增加展开状态。展开状态保持可拖动宽度并提供收起按钮；收起状态使用独立的窄面板 ID，只显示展开按钮，避免覆盖展开面板保存的宽度。中央歌词面板由 egui 自动取得剩余区域。

## 验证

使用包含相同时间戳中英文行的 `GQ.lrc` 验证聚合逻辑；运行单元测试、严格 Clippy 和 release 构建；在 150% DPI 下检查展开及收起两种布局，确认歌词、底栏和侧栏无越界或重叠。
